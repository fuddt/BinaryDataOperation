今回は、コード内で `patchData` を手作りするのをやめます。

# Step 1：patch.bin の中身を決める

今回は `player_name` を `TARO` に変更したいとします。

`player_name` は16バイト固定長なので、`patch.bin` も16バイトにします。

中身はこれです。

```text
54 41 52 4F 00 00 00 00 00 00 00 00 00 00 00 00
```

これはASCII/UTF-8で見ると：

```text
T A R O \0 \0 \0 ...
```

です。

重要なのは、**TAROの4バイトだけでは足りない**ということです。

`player_name` フィールドは16バイト固定なので、`patch.bin` も16バイトにします。

# Step 2：patch.bin を作るコード

まず、教材用に `player_name_patch.bin` を作るプログラムを書きます。

```cpp
#include <fstream>
#include <vector>
#include <cstdint>
#include <cstring>
#include <string>
#include <stdexcept>

// 16バイト固定長の player_name 用 patch.bin を作る関数
void createPlayerNamePatchBin(const std::string& filePath,
                              const std::string& playerName)
{
    // player_name は16バイト固定長という仕様
    constexpr size_t PLAYER_NAME_SIZE = 16;

    // 16バイト分の領域を用意する
    // 初期値は全部 0x00 にしておく
    std::vector<uint8_t> data(PLAYER_NAME_SIZE, 0x00);

    // 入力された名前が16バイトを超える場合は危険なので拒否する
    // ここでは簡単のため、文字数 = バイト数として扱う
    // 日本語などマルチバイト文字を扱う場合は別途注意が必要
    if (playerName.size() > PLAYER_NAME_SIZE)
    {
        throw std::runtime_error("playerName is too long");
    }

    // data の先頭に playerName のバイト列をコピーする
    // 余った部分は 0x00 のまま残る
    std::memcpy(data.data(),
                playerName.data(),
                playerName.size());

    // バイナリモードでファイルを開く
    std::ofstream file(filePath, std::ios::binary);

    if (!file)
    {
        throw std::runtime_error("failed to open output file");
    }

    // 16バイトをそのままファイルに書く
    file.write(reinterpret_cast<const char*>(data.data()),
               static_cast<std::streamsize>(data.size()));

    if (!file)
    {
        throw std::runtime_error("failed to write patch bin");
    }
}

int main()
{
    createPlayerNamePatchBin("player_name_patch.bin", "TARO");
    return 0;
}
```

これを実行すると、カレントディレクトリに：

```text
player_name_patch.bin
```

ができます。

# Step 4：patch.bin を読み込む関数

次に、`.bin` を読み込む関数です。

```cpp
#include <fstream>
#include <vector>
#include <cstdint>
#include <string>
#include <stdexcept>

// .binファイルをバイト列として読み込む関数
std::vector<uint8_t> readBinaryFile(const std::string& filePath)
{
    // バイナリモードで開く
    std::ifstream file(filePath, std::ios::binary);

    if (!file)
    {
        throw std::runtime_error("failed to open binary file");
    }

    // ファイル末尾へ移動してサイズを調べる
    file.seekg(0, std::ios::end);
    std::streamsize fileSize = file.tellg();

    // ファイル先頭へ戻す
    file.seekg(0, std::ios::beg);

    if (fileSize < 0)
    {
        throw std::runtime_error("failed to get file size");
    }

    // ファイルサイズ分のバッファを用意する
    std::vector<uint8_t> data(static_cast<size_t>(fileSize));

    // ファイル内容をそのまま読み込む
    // ifstream::read は char* を要求するため、uint8_t* を char* に変換する
    if (!file.read(reinterpret_cast<char*>(data.data()), fileSize))
    {
        throw std::runtime_error("failed to read binary file");
    }

    return data;
}
```

ポイントは2つです。

```cpp
std::ios::binary
```

これは必須です。

そして：

```cpp
reinterpret_cast<char*>(data.data())
```

これは、`ifstream::read` が `char*` を要求するために必要です。

# Step 5：patch.bin を使って player_name を差し替える完成コード

では、前回の教材コードとつなげます。

```cpp
#include <iostream>
#include <fstream>
#include <cstdint>
#include <cstddef>
#include <cstring>
#include <vector>
#include <iomanip>
#include <string>
#include <stdexcept>

// buffer内のメタデータ領域を表す構造体
struct MetadataRegion
{
    size_t baseOffset;  // buffer全体の先頭からmetadata領域が始まる位置
    size_t size;        // metadata領域全体のサイズ
};

// metadata領域内の1フィールドを表す構造体
struct FieldDescriptor
{
    size_t offset;      // metadata領域の先頭から見た相対offset
    size_t size;        // フィールドのバイト数
};

// 指定範囲のバイト列を16進数で表示する関数
void printBytes(const uint8_t* buffer, size_t start, size_t size)
{
    for (size_t i = 0; i < size; ++i)
    {
        std::cout
            << std::hex
            << std::setw(2)
            << std::setfill('0')
            << static_cast<int>(buffer[start + i])
            << " ";
    }

    std::cout << std::dec << std::endl;
}

// .binファイルをバイト列として読み込む関数
std::vector<uint8_t> readBinaryFile(const std::string& filePath)
{
    // バイナリモードで開く
    std::ifstream file(filePath, std::ios::binary);

    if (!file)
    {
        throw std::runtime_error("failed to open binary file");
    }

    // ファイルサイズを取得する
    file.seekg(0, std::ios::end);
    std::streamsize fileSize = file.tellg();
    file.seekg(0, std::ios::beg);

    if (fileSize < 0)
    {
        throw std::runtime_error("failed to get file size");
    }

    // ファイルサイズ分のバッファを用意する
    std::vector<uint8_t> data(static_cast<size_t>(fileSize));

    // 中身をそのまま読み込む
    if (!file.read(reinterpret_cast<char*>(data.data()), fileSize))
    {
        throw std::runtime_error("failed to read binary file");
    }

    return data;
}

// metadata内の特定フィールドだけを、patchDataで差し替える関数
bool patchMetadataField(
    uint8_t* buffer,
    size_t bufferSize,
    const MetadataRegion& metadata,
    const FieldDescriptor& field,
    const std::vector<uint8_t>& patchData
)
{
    // bufferが存在するか確認する
    if (buffer == nullptr)
    {
        return false;
    }

    // patchDataのサイズが、差し替え先フィールドサイズと一致するか確認する
    if (patchData.size() != field.size)
    {
        return false;
    }

    // fieldがmetadata領域の中に収まっているか確認する
    if (field.offset > metadata.size)
    {
        return false;
    }

    if (field.size > metadata.size - field.offset)
    {
        return false;
    }

    // metadata領域がbuffer全体の中に収まっているか確認する
    if (metadata.baseOffset > bufferSize)
    {
        return false;
    }

    if (metadata.size > bufferSize - metadata.baseOffset)
    {
        return false;
    }

    // metadata内の相対offsetを、buffer全体の絶対offsetに変換する
    const size_t absoluteOffset = metadata.baseOffset + field.offset;

    // 書き換え先がbufferの範囲内か最終確認する
    if (absoluteOffset > bufferSize)
    {
        return false;
    }

    if (field.size > bufferSize - absoluteOffset)
    {
        return false;
    }

    // 対象フィールドだけを上書きする
    std::memcpy(buffer + absoluteOffset,
                patchData.data(),
                patchData.size());

    return true;
}

int main()
{
    try
    {
        // 教材用のbuffer。
        // 実際には巨大ファイルから読み込まれたメモリ領域だと思ってください。
        std::vector<uint8_t> buffer(1200, 0x00);

        // metadata領域は buffer の1000バイト目から64バイト分とする
        MetadataRegion metadata{
            1000, // baseOffset
            64    // size
        };

        // player_name は metadata領域の4バイト目から16バイト分とする
        FieldDescriptor playerNameField{
            4,   // metadata内の相対offset
            16   // 固定長サイズ
        };

        // 実際にbuffer上でplayer_nameが始まる位置
        const size_t playerNameAbsoluteOffset =
            metadata.baseOffset + playerNameField.offset;

        // 初期値として "OLD_NAME" を入れておく
        const char* oldName = "OLD_NAME";

        std::memcpy(buffer.data() + playerNameAbsoluteOffset,
                    oldName,
                    std::strlen(oldName));

        std::cout << "Before patch:" << std::endl;
        printBytes(buffer.data(),
                   playerNameAbsoluteOffset,
                   playerNameField.size);

        // ここで実際に patch.bin を読み込む
        std::vector<uint8_t> patchData =
            readBinaryFile("player_name_patch.bin");

        // patch.bin の中身を確認する
        std::cout << "Patch data:" << std::endl;
        printBytes(patchData.data(),
                   0,
                   patchData.size());

        // player_nameフィールドだけ差し替える
        bool ok = patchMetadataField(buffer.data(),
                                     buffer.size(),
                                     metadata,
                                     playerNameField,
                                     patchData);

        std::cout << "Patch result: " << ok << std::endl;

        std::cout << "After patch:" << std::endl;
        printBytes(buffer.data(),
                   playerNameAbsoluteOffset,
                   playerNameField.size);
    }
    catch (const std::exception& e)
    {
        std::cerr << "error: " << e.what() << std::endl;
        return 1;
    }

    return 0;
}
```

# ここで何が起きているか

処理はこの順番です。

```text
1. buffer を作る
2. buffer の metadata 領域に OLD_NAME が入っている状態にする
3. player_name_patch.bin を読む
4. patch.bin のサイズが player_name の16バイトと一致するか確認する
5. buffer[1004] から16バイトだけ上書きする
6. buffer を確認すると TARO に変わっている
```

重要なのは、**buffer全体を作り直していない**ことです。

また、**metadata全体64バイトも上書きしていません**。

書き換えているのは：

```text
player_name の16バイトだけ
```

です。

# 失敗パターンも見る

ここが実務では重要です。

例えば `player_name_patch.bin` が4バイトしかない場合：

```text
54 41 52 4F
```

つまり `TARO` だけ。

このとき：

```cpp
if (patchData.size() != field.size)
{
    return false;
}
```

に引っかかります。

なぜなら：

```text
patchData.size() = 4
field.size       = 16
```

だからです。

このチェックは必要です。

ここを通してしまうと、次のような曖昧な仕様になります。

```text
TAROだけ上書きして残り12バイトは古い値を残すのか？
TARO以降を0埋めするのか？
TARO以降をスペース埋めするのか？
```

これは勝手に決めるべきではありません。

今回の授業では、固定長フィールドなので：

```text
patch.bin はフィールドサイズと完全一致
```

をルールにします。

これは堅い設計です。

# では「TAROだけの4バイトbin」を許可したい場合は？

その場合は、仕様を変える必要があります。

例えば：

```text
短い文字列patchを許可する
余りは0x00で埋める
```

という仕様にします。

この場合、`.bin` というより「文字列入力」に近いです。

実装もこう変わります。

```cpp
// patchDataがfield.size以下なら許可し、残りを0x00で埋める
std::vector<uint8_t> normalizeFixedSizePatch(
    const std::vector<uint8_t>& patchData,
    size_t fieldSize
)
{
    if (patchData.size() > fieldSize)
    {
        throw std::runtime_error("patchData is too large");
    }

    std::vector<uint8_t> normalized(fieldSize, 0x00);

    std::memcpy(normalized.data(),
                patchData.data(),
                patchData.size());

    return normalized;
}
```

ただし、最初はこれはやらない方がいいです。

理由は、暗黙の変換が増えるからです。

最初は：

```text
patch.bin のサイズは field.size と一致
```

で進める方が安全です。

# 今回の授業のまとめ

今回で、`.bin` を使った差し替えの基本形は完成です。

構造はこうです。

```text
player_name_patch.bin
↓
readBinaryFile()
↓
std::vector<uint8_t> patchData
↓
patchMetadataField()
↓
buffer[metadata.baseOffset + field.offset] から field.size バイトだけ上書き
```

このとき必要な安全条件は：

```text
1. patch.bin が読める
2. patch.bin のサイズが field.size と一致する
3. metadata領域がbuffer内に収まっている
4. fieldがmetadata領域内に収まっている
5. 書き換え先がbuffer範囲内に収まっている
```

ここまでできれば、**一部フィールドだけを `.bin` で差し替える**という基本設計は理解できています。

次に進むなら、次は **「複数フィールドを差し替える場合」** をやるのが自然です。
例えば `player_name` と `stage_id` を、それぞれ別の `.bin` で差し替えるパターンです。
