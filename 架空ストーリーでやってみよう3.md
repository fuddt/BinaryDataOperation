では次は **複数フィールドを差し替える場合** を授業します。

前回までは、1つのフィールドだけでした。

```text
player_name だけ差し替える
```

今回はこうします。

```text
player_name を差し替える
stage_id も差し替える
```

しかも、それぞれ別の `.bin` ファイルから読みます。

---

# 今回のシナリオ

前回と同じく、架空のゲームリプレイデータです。

```text
buffer全体

[ 操作ログ本体 ][ metadata領域 ][ その他データ ]
```

metadata領域は、bufferの1000バイト目から64バイト分とします。

```text
metadataBaseOffset = 1000
metadataSize       = 64
```

metadataのレイアウトはこれです。

```text
metadata領域 64 bytes

offset 0   size 4    replay_version
offset 4   size 16   player_name
offset 20  size 4    stage_id
offset 24  size 8    recorded_at
offset 32  size 32   reserved
```

今回差し替えるのはこの2つ。

```text
player_name: offset 4,  size 16
stage_id:    offset 20, size 4
```

実際のbuffer上の位置はこうなります。

```text
player_name = metadataBaseOffset + 4
            = 1000 + 4
            = 1004

stage_id    = metadataBaseOffset + 20
            = 1000 + 20
            = 1020
```

---

# 今回使うpatchファイル

今回は2つの `.bin` を使う設定にします。

```text
player_name_patch.bin
stage_id_patch.bin
```

それぞれの役割はこうです。

```text
player_name_patch.bin
→ player_name 16バイト分の差し替えデータ

stage_id_patch.bin
→ stage_id 4バイト分の差し替えデータ
```

重要なのは、**それぞれの `.bin` は差し替え先フィールドのサイズと完全一致する**というルールです。

```text
player_name_patch.bin size = 16 bytes
stage_id_patch.bin size    = 4 bytes
```

---

# まずpatch対象を表す構造を作る

1フィールドだけなら、関数に `FieldDescriptor` と `patchData` を直接渡せばよかった。

でも複数になると、こういう「差し替え指示」が欲しくなります。

```cpp
struct PatchJob
{
    std::string fieldName;
    std::string binFilePath;
};
```

意味はこうです。

```text
fieldName   = どのフィールドを差し替えるか
binFilePath = どの .bin を使うか
```

例えば：

```cpp
PatchJob{
    "player_name",
    "player_name_patch.bin"
}
```

これは：

```text
player_name フィールドを player_name_patch.bin で差し替える
```

という意味です。

---

# layoutをmapで持つ

複数フィールドを扱う場合、毎回 `FieldDescriptor` を手で渡すより、フィールド名から引けるようにします。

```cpp
using MetadataLayout = std::unordered_map<std::string, FieldDescriptor>;
```

こうすると：

```cpp
layout["player_name"]
layout["stage_id"]
```

で、それぞれのoffset/sizeを取得できます。

---

# 完成コード

今回は、以下を全部まとめます。

1. `player_name_patch.bin` を作る
2. `stage_id_patch.bin` を作る
3. buffer上に古い値を入れる
4. 複数のPatchJobを順番に適用する
5. 差し替え後のbufferを確認する

```cpp
#include <iostream>
#include <fstream>
#include <cstdint>
#include <cstddef>
#include <cstring>
#include <vector>
#include <iomanip>
#include <string>
#include <stdexcept>
#include <unordered_map>

// buffer内のmetadata領域を表す
struct MetadataRegion
{
    size_t baseOffset;  // buffer全体の先頭からmetadata領域が始まる位置
    size_t size;        // metadata領域全体のサイズ
};

// metadata領域内の1フィールドを表す
struct FieldDescriptor
{
    size_t offset;      // metadata領域の先頭から見た相対offset
    size_t size;        // フィールドのバイト数
};

// フィールド名からoffset/sizeを引けるようにするlayout表
using MetadataLayout = std::unordered_map<std::string, FieldDescriptor>;

// 1件分のpatch指示
struct PatchJob
{
    std::string fieldName;    // 差し替え対象フィールド名
    std::string binFilePath;  // 差し替えデータが入った.binファイル
};

// 指定範囲のバイト列を16進数で表示する関数
void printBytes(const uint8_t* buffer, size_t start, size_t size)
{
    for (size_t i = 0; i < size; ++i)
    {
        std::cout
            << std::hex
            << std::setw(2)
            << std::setfill('0')
            << static_cast<int>(buffer[start + i])
            << " ";
    }

    std::cout << std::dec << std::endl;
}

// .binファイルをそのままバイト列として読み込む
std::vector<uint8_t> readBinaryFile(const std::string& filePath)
{
    std::ifstream file(filePath, std::ios::binary);

    if (!file)
    {
        throw std::runtime_error("failed to open binary file: " + filePath);
    }

    file.seekg(0, std::ios::end);
    std::streamsize fileSize = file.tellg();
    file.seekg(0, std::ios::beg);

    if (fileSize < 0)
    {
        throw std::runtime_error("failed to get file size: " + filePath);
    }

    std::vector<uint8_t> data(static_cast<size_t>(fileSize));

    if (!file.read(reinterpret_cast<char*>(data.data()), fileSize))
    {
        throw std::runtime_error("failed to read binary file: " + filePath);
    }

    return data;
}

// バイト列を.binファイルとして書き出す
void writeBinaryFile(const std::string& filePath,
                     const std::vector<uint8_t>& data)
{
    std::ofstream file(filePath, std::ios::binary);

    if (!file)
    {
        throw std::runtime_error("failed to open output file: " + filePath);
    }

    file.write(reinterpret_cast<const char*>(data.data()),
               static_cast<std::streamsize>(data.size()));

    if (!file)
    {
        throw std::runtime_error("failed to write binary file: " + filePath);
    }
}

// 固定長文字列のpatchデータを作る
std::vector<uint8_t> makeFixedStringPatch(const std::string& text,
                                          size_t fieldSize)
{
    // fieldSizeバイト分を0x00で初期化する
    std::vector<uint8_t> data(fieldSize, 0x00);

    // 今回は教材なので、文字数=バイト数として扱う
    // 日本語などのマルチバイト文字はここでは扱わない
    if (text.size() > fieldSize)
    {
        throw std::runtime_error("text is too long");
    }

    // 先頭から文字列のバイト列をコピーする
    std::memcpy(data.data(),
                text.data(),
                text.size());

    return data;
}

// uint32_tをlittle endianの4バイトにする
std::vector<uint8_t> makeUInt32LittleEndianPatch(uint32_t value)
{
    std::vector<uint8_t> data(4, 0x00);

    // little endianとして、下位バイトから順に格納する
    data[0] = static_cast<uint8_t>(value & 0xFF);
    data[1] = static_cast<uint8_t>((value >> 8) & 0xFF);
    data[2] = static_cast<uint8_t>((value >> 16) & 0xFF);
    data[3] = static_cast<uint8_t>((value >> 24) & 0xFF);

    return data;
}

// metadata内の特定フィールドだけをpatchDataで差し替える
bool patchMetadataField(
    uint8_t* buffer,
    size_t bufferSize,
    const MetadataRegion& metadata,
    const FieldDescriptor& field,
    const std::vector<uint8_t>& patchData
)
{
    if (buffer == nullptr)
    {
        return false;
    }

    // patchDataのサイズはフィールドサイズと完全一致させる
    if (patchData.size() != field.size)
    {
        return false;
    }

    // fieldがmetadata領域内に収まるか確認する
    if (field.offset > metadata.size)
    {
        return false;
    }

    if (field.size > metadata.size - field.offset)
    {
        return false;
    }

    // metadata領域がbuffer内に収まるか確認する
    if (metadata.baseOffset > bufferSize)
    {
        return false;
    }

    if (metadata.size > bufferSize - metadata.baseOffset)
    {
        return false;
    }

    // metadata内相対offsetをbuffer全体の絶対offsetに変換する
    const size_t absoluteOffset = metadata.baseOffset + field.offset;

    // 書き換え先がbuffer内に収まるか確認する
    if (absoluteOffset > bufferSize)
    {
        return false;
    }

    if (field.size > bufferSize - absoluteOffset)
    {
        return false;
    }

    // 対象フィールドだけを上書きする
    std::memcpy(buffer + absoluteOffset,
                patchData.data(),
                patchData.size());

    return true;
}

// 1件のPatchJobを適用する
bool applyPatchJob(
    uint8_t* buffer,
    size_t bufferSize,
    const MetadataRegion& metadata,
    const MetadataLayout& layout,
    const PatchJob& job
)
{
    // 1. fieldNameがlayoutに存在するか確認する
    auto it = layout.find(job.fieldName);

    if (it == layout.end())
    {
        std::cerr << "unknown field: " << job.fieldName << std::endl;
        return false;
    }

    const FieldDescriptor& field = it->second;

    // 2. .binファイルを読み込む
    std::vector<uint8_t> patchData;

    try
    {
        patchData = readBinaryFile(job.binFilePath);
    }
    catch (const std::exception& e)
    {
        std::cerr << e.what() << std::endl;
        return false;
    }

    // 3. 読み込んだpatchDataを対象フィールドへ適用する
    bool ok = patchMetadataField(
        buffer,
        bufferSize,
        metadata,
        field,
        patchData
    );

    if (!ok)
    {
        std::cerr
            << "failed to patch field: "
            << job.fieldName
            << std::endl;
        return false;
    }

    return true;
}

// 複数のPatchJobを順番に適用する
bool applyPatchJobs(
    uint8_t* buffer,
    size_t bufferSize,
    const MetadataRegion& metadata,
    const MetadataLayout& layout,
    const std::vector<PatchJob>& jobs
)
{
    for (const PatchJob& job : jobs)
    {
        bool ok = applyPatchJob(
            buffer,
            bufferSize,
            metadata,
            layout,
            job
        );

        if (!ok)
        {
            return false;
        }
    }

    return true;
}

int main()
{
    try
    {
        // 教材用にpatch用.binを作る
        // player_nameは16バイト固定長
        writeBinaryFile(
            "player_name_patch.bin",
            makeFixedStringPatch("TARO", 16)
        );

        // stage_idはuint32 little endianで4バイト
        // 例として stage_id = 42 にする
        writeBinaryFile(
            "stage_id_patch.bin",
            makeUInt32LittleEndianPatch(42)
        );

        // 実際には巨大ファイルから読み込まれたbufferだと思ってください
        std::vector<uint8_t> buffer(1200, 0x00);

        // metadata領域はbufferの1000バイト目から64バイト分
        MetadataRegion metadata{
            1000,
            64
        };

        // metadataのlayout定義
        MetadataLayout layout{
            {"replay_version", {0, 4}},
            {"player_name",    {4, 16}},
            {"stage_id",       {20, 4}},
            {"recorded_at",    {24, 8}},
            {"reserved",       {32, 32}}
        };

        // 初期状態としてplayer_nameにOLD_NAMEを入れる
        {
            const size_t offset =
                metadata.baseOffset + layout["player_name"].offset;

            const char* oldName = "OLD_NAME";

            std::memcpy(buffer.data() + offset,
                        oldName,
                        std::strlen(oldName));
        }

        // 初期状態としてstage_idに1を入れる
        {
            const size_t offset =
                metadata.baseOffset + layout["stage_id"].offset;

            std::vector<uint8_t> oldStage =
                makeUInt32LittleEndianPatch(1);

            std::memcpy(buffer.data() + offset,
                        oldStage.data(),
                        oldStage.size());
        }

        std::cout << "Before patch - player_name:" << std::endl;
        printBytes(buffer.data(),
                   metadata.baseOffset + layout["player_name"].offset,
                   layout["player_name"].size);

        std::cout << "Before patch - stage_id:" << std::endl;
        printBytes(buffer.data(),
                   metadata.baseOffset + layout["stage_id"].offset,
                   layout["stage_id"].size);

        // 複数のpatch指示
        std::vector<PatchJob> jobs{
            {"player_name", "player_name_patch.bin"},
            {"stage_id",    "stage_id_patch.bin"}
        };

        bool ok = applyPatchJobs(
            buffer.data(),
            buffer.size(),
            metadata,
            layout,
            jobs
        );

        std::cout << "Patch result: " << ok << std::endl;

        std::cout << "After patch - player_name:" << std::endl;
        printBytes(buffer.data(),
                   metadata.baseOffset + layout["player_name"].offset,
                   layout["player_name"].size);

        std::cout << "After patch - stage_id:" << std::endl;
        printBytes(buffer.data(),
                   metadata.baseOffset + layout["stage_id"].offset,
                   layout["stage_id"].size);
    }
    catch (const std::exception& e)
    {
        std::cerr << "error: " << e.what() << std::endl;
        return 1;
    }

    return 0;
}
```

# 今回の重要ポイント

今回から、処理が少し設計っぽくなっています。

1フィールドだけなら、こうでした。

```text
FieldDescriptor + patchData
```

複数フィールドになると、こうなります。

```text
MetadataLayout
+
PatchJob一覧
```

つまり、

```text
layout:
field名 → offset/size

patch job:
field名 → binファイル
```

に分けています。

これがかなり重要です。

---

# layoutとpatch jobを分ける理由

`layout` は構造の定義です。

```text
player_name は metadata内 offset 4, size 16
stage_id は metadata内 offset 20, size 4
```

`patch job` は今回の変更指示です。

```text
player_name を player_name_patch.bin で差し替える
stage_id を stage_id_patch.bin で差し替える
```

この2つは役割が違います。

混ぜると設計が弱くなります。

---

# 今回の設計図

```text
buffer
  ↓
metadata region
  baseOffset = 1000
  size = 64

layout
  player_name -> offset 4,  size 16
  stage_id    -> offset 20, size 4

patch jobs
  player_name -> player_name_patch.bin
  stage_id    -> stage_id_patch.bin

処理
  各jobについて:
    1. fieldNameからlayoutを引く
    2. binを読む
    3. binサイズとfieldサイズを比較
    4. absoluteOffset = metadata.baseOffset + field.offset
    5. memcpyで上書き
```

---

# 今回のコードの弱点

ただし、今回のコードにはまだ実務上の弱点があります。

それは：

```text
途中までpatch成功してから、後続patchで失敗する可能性がある
```

例えば：

```text
player_name は成功
stage_id は失敗
```

となると、bufferは中途半端に変更された状態になります。

これは危険です。

---

# 次に学ぶべきこと

次は **validate all before apply** をやります。

つまり：

```text
悪い流れ:
1件目を適用
2件目を適用
3件目で失敗
→ bufferが中途半端に変更済み

良い流れ:
全patchを先に検証
全部OKなら適用
1件でもNGなら何も変更しない
```

これをやると、設計が一段階まともになります。
