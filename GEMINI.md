# BinaryDataOperation プロジェクト概要

このプロジェクトは、C++ を使用してバイナリデータの構造（レイアウト）を理解し、安全にデータを操作（部分的なパッチ適用など）するための学習および設計用リポジトリです。

## プロジェクトの目的
- メモリの仕組み（バイト列、エンディアン、アドレス、オフセット）の理解。
- バイナリデータのレイアウト設計（オフセットとサイズによる定義）の習得。
- フォーマットの差異を吸収しつつ、特定のフィールドのみを安全に更新する「Partial Binary Patch」の設計と実装。

## 主な技術スタック
- **言語:** C++20
- **ビルドシステム:** CMake (4.1以上)
- **開発環境:** CLion 等の C++ IDE

## ディレクトリ構造と主要ファイル
- `main.cpp`: 実験的なコードを実行するためのエントリポイント。
- `CMakeLists.txt`: ビルド設定ファイル。
- **学習ドキュメント (Markdown):**
    - `1._メモリとは何か.md`: Phase 1。メモリの基本、バイト列、変数の保存方法。
    - `2_バッファとは何か？.md`: バッファの概念と操作。
    - `レイアウトとは.md` / `レイアウトテーブル設計とは.md`: データの配置図（地図）としてのレイアウト定義。
    - `partial_binary_patch設計.md`: Phase 5。特定の範囲だけを安全に差し替える仕組みの設計。
    - `パッチの実践例.md`: 実際のコードへの適用例。
    - `架空ストーリーでやってみよう[1-4].md`: 具体的なシナリオに基づいた実践演習。

## ビルドと実行方法
### ビルド
```bash
mkdir cmake-build-debug
cmake -S . -B cmake-build-debug
cmake --build cmake-build-debug
```

### 実行
```bash
./cmake-build-debug/BinaryDataOperation
```

## 開発の進め方
1. 各ドキュメント（Phase 1, 2, ...）を読み、バイナリ操作の理論を理解する。
2. `main.cpp` に実験用コードを書き、メモリの挙動を実際に確認する。
3. レイアウトテーブル（`std::unordered_map<FieldID, FieldDescriptor>`）を用いた設計を実装し、フォーマット差異に強いバイナリ操作ライブラリを構築する。

## 注意事項
- `std::memcpy` を使用した直接的なメモリ操作を行うため、バッファオーバーランや `nullptr` チェックなど、安全性に細心の注意を払うこと。
- 実務的な設計では、境界チェック、サイズ一致チェック、フォーマット判定などのバリデーションを必須とする。
