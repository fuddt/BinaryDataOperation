Step3：メモリの中身を見る

ここが重要な実験

次のコードを書いてください：

```cpp
#include <iostream>
#include <cstdint>

// メモリの中身を1バイトずつ表示する関数
void printBytes(void* ptr, size_t size)
{
// void* は型情報がないので uint8_t* に変換
uint8_t* p = static_cast<uint8_t*>(ptr);

    for(size_t i = 0; i < size; i++)
    {
        std::cout << (int)p[i] << " ";
    }

    std::cout << std::endl;
}

int main()
{
int x = 100;

    std::cout << "sizeof(x): "
              << sizeof(x)
              << std::endl;

    printBytes(&x, sizeof(x));
}
```

ここで何が起きているか説明する

この行：

&x

意味：

xのメモリアドレス

この関数：

printBytes(&x, sizeof(x))

意味：

xの保存されているメモリを観察する
なぜ uint8_t* に変換する？

ここが核心

void* ptr

は：

型なしアドレス

なので：

uint8_t*

に変換する

理由：

1バイト単位で読むため
実行するとどうなるか

たぶんこうなる：

100 0 0 0

または：

0 0 0 100

どちらか

これは：

エンディアン

の違い

（後でやる）

---

# 1バイトとは何か

1バイトは

```
8ビット
```

です。

ビットとは

```
0 か 1
```

のどちらかだけを持つ最小単位です。

つまり：

```
1ビット = 2通り
```

---

# 8ビットなら何通り？

8ビットなので：

```
2 × 2 × 2 × 2 × 2 × 2 × 2 × 2
= 2^8
= 256通り
```

表現できます。

---

# 256通りあるなら最大値はいくつ？

ここが重要。

256通りということは：

```
0〜255
```

まで表現できます。

理由：

```
0を含むから
```

つまり：

```
全部で256個
```

---

# 具体的に書くとこうなる

8ビット：

```
00000000 = 0
00000001 = 1
00000010 = 2
...
11111111 = 255
```

つまり：

```
最大値 = 255
```

です。

---

# では256はどう表現される？

ここがさっきの話とつながります。

256は：

```
00000001 00000000
```

になります。

つまり：

```
2バイト必要
```

です。

理由：

1バイトでは

```
11111111
```

までしか入らないからです。

---

# 実際にメモリではこう並ぶ

例えば：

```
uint8_t x = 255;
```

なら

```
11111111
```

ですが

```
uint16_t x = 256;
```

なら

```
00000001 00000000
```

になります。
