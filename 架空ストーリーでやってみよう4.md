では次は **validate all before apply** をやります。

今回のテーマはこれです。

> 複数フィールドを差し替えるとき、いきなり書き換えず、先に全件検証して、全部OKなら初めてbufferを書き換える。

これは実務ではかなり重要です。

---

# なぜ必要か

前回のコードでは、複数patchを順番に適用していました。

```text
1. player_name をpatch
2. stage_id をpatch
3. recorded_at をpatch
```

このとき、もし3件目で失敗するとどうなるか。

```text
player_name → 変更済み
stage_id    → 変更済み
recorded_at → 失敗
```

つまり、bufferが**中途半端に変更された状態**になります。

これは危険です。

---

# 悪い流れ

```text
buffer初期状態
↓
player_name patch 成功
↓
stage_id patch 成功
↓
recorded_at patch 失敗
↓
中途半端に変更されたbufferが残る
```

これが一番避けたい。

---

# 良い流れ

```text
patch job一覧を受け取る
↓
全patchについて事前検証する
↓
1件でもNGなら何も変更しない
↓
全部OKならまとめて適用する
```

つまり：

```text
validate
↓
apply
```

の2段階にする。

---

# 今回の設計

前回の構造はこれでした。

```cpp
struct PatchJob
{
    std::string fieldName;
    std::string binFilePath;
};
```

でも、validateとapplyを分けるには、`.bin` を読んだ結果も保持したい。

そこで、検証済みpatchを表す構造体を追加します。

```cpp
struct PreparedPatch
{
    std::string fieldName;
    FieldDescriptor field;
    std::vector<uint8_t> patchData;
};
```

意味はこうです。

```text
PreparedPatch
=
field名も分かっている
offset/sizeも分かっている
binも読めている
サイズも検証済み
```

つまり、apply段階ではもう失敗しにくい状態にしておく。

---

# 全体の流れ

```text
PatchJob一覧
↓
preparePatches()
  - fieldName がlayoutに存在するか確認
  - binファイルを読めるか確認
  - patchData.size と field.size が一致するか確認
  - fieldがmetadata領域内に収まるか確認
  - metadata領域がbuffer内に収まるか確認
↓
PreparedPatch一覧
↓
applyPreparedPatches()
  - memcpyで実際にbufferを書き換える
```

---

# 完成コード

前回のコードを、**全件検証してから適用する形**にします。

```cpp
#include <iostream>
#include <fstream>
#include <cstdint>
#include <cstddef>
#include <cstring>
#include <vector>
#include <iomanip>
#include <string>
#include <stdexcept>
#include <unordered_map>

// buffer内のmetadata領域を表す
struct MetadataRegion
{
    size_t baseOffset;  // buffer全体の先頭からmetadata領域が始まる位置
    size_t size;        // metadata領域全体のサイズ
};

// metadata領域内の1フィールドを表す
struct FieldDescriptor
{
    size_t offset;      // metadata領域の先頭から見た相対offset
    size_t size;        // フィールドのバイト数
};

// フィールド名からoffset/sizeを引けるようにするlayout表
using MetadataLayout = std::unordered_map<std::string, FieldDescriptor>;

// 外から与えられるpatch指示
struct PatchJob
{
    std::string fieldName;    // 差し替え対象フィールド名
    std::string binFilePath;  // 差し替えデータが入った.binファイル
};

// 検証済みpatch
// ここまで来たら、bin読み込み済み・サイズ確認済み・field確認済み
struct PreparedPatch
{
    std::string fieldName;          // ログやデバッグ用
    FieldDescriptor field;          // 差し替え先のoffset/size
    std::vector<uint8_t> patchData; // 実際に書き込むバイト列
};

// 指定範囲のバイト列を16進数で表示する関数
void printBytes(const uint8_t* buffer, size_t start, size_t size)
{
    for (size_t i = 0; i < size; ++i)
    {
        std::cout
            << std::hex
            << std::setw(2)
            << std::setfill('0')
            << static_cast<int>(buffer[start + i])
            << " ";
    }

    std::cout << std::dec << std::endl;
}

// .binファイルをそのままバイト列として読み込む
std::vector<uint8_t> readBinaryFile(const std::string& filePath)
{
    std::ifstream file(filePath, std::ios::binary);

    if (!file)
    {
        throw std::runtime_error("failed to open binary file: " + filePath);
    }

    file.seekg(0, std::ios::end);
    std::streamsize fileSize = file.tellg();
    file.seekg(0, std::ios::beg);

    if (fileSize < 0)
    {
        throw std::runtime_error("failed to get file size: " + filePath);
    }

    std::vector<uint8_t> data(static_cast<size_t>(fileSize));

    if (!file.read(reinterpret_cast<char*>(data.data()), fileSize))
    {
        throw std::runtime_error("failed to read binary file: " + filePath);
    }

    return data;
}

// バイト列を.binファイルとして書き出す
void writeBinaryFile(const std::string& filePath,
                     const std::vector<uint8_t>& data)
{
    std::ofstream file(filePath, std::ios::binary);

    if (!file)
    {
        throw std::runtime_error("failed to open output file: " + filePath);
    }

    file.write(reinterpret_cast<const char*>(data.data()),
               static_cast<std::streamsize>(data.size()));

    if (!file)
    {
        throw std::runtime_error("failed to write binary file: " + filePath);
    }
}

// 固定長文字列のpatchデータを作る
std::vector<uint8_t> makeFixedStringPatch(const std::string& text,
                                          size_t fieldSize)
{
    std::vector<uint8_t> data(fieldSize, 0x00);

    // 教材用なので、ASCII前提として文字数=バイト数で扱う
    if (text.size() > fieldSize)
    {
        throw std::runtime_error("text is too long");
    }

    std::memcpy(data.data(),
                text.data(),
                text.size());

    return data;
}

// uint32_tをlittle endianの4バイトにする
std::vector<uint8_t> makeUInt32LittleEndianPatch(uint32_t value)
{
    std::vector<uint8_t> data(4, 0x00);

    // little endianなので、下位バイトから順に格納する
    data[0] = static_cast<uint8_t>(value & 0xFF);
    data[1] = static_cast<uint8_t>((value >> 8) & 0xFF);
    data[2] = static_cast<uint8_t>((value >> 16) & 0xFF);
    data[3] = static_cast<uint8_t>((value >> 24) & 0xFF);

    return data;
}

// fieldがmetadata領域内に収まっているか確認する
bool isFieldInsideMetadata(const MetadataRegion& metadata,
                           const FieldDescriptor& field)
{
    if (field.offset > metadata.size)
    {
        return false;
    }

    if (field.size > metadata.size - field.offset)
    {
        return false;
    }

    return true;
}

// metadata領域がbuffer内に収まっているか確認する
bool isMetadataInsideBuffer(size_t bufferSize,
                            const MetadataRegion& metadata)
{
    if (metadata.baseOffset > bufferSize)
    {
        return false;
    }

    if (metadata.size > bufferSize - metadata.baseOffset)
    {
        return false;
    }

    return true;
}

// fieldの絶対offsetがbuffer内に収まっているか確認する
bool isPatchDestinationInsideBuffer(size_t bufferSize,
                                    const MetadataRegion& metadata,
                                    const FieldDescriptor& field)
{
    // metadata自体がbuffer外なら当然NG
    if (!isMetadataInsideBuffer(bufferSize, metadata))
    {
        return false;
    }

    // fieldがmetadata外ならNG
    if (!isFieldInsideMetadata(metadata, field))
    {
        return false;
    }

    const size_t absoluteOffset = metadata.baseOffset + field.offset;

    if (absoluteOffset > bufferSize)
    {
        return false;
    }

    if (field.size > bufferSize - absoluteOffset)
    {
        return false;
    }

    return true;
}

// PatchJob一覧を事前検証し、適用可能なPreparedPatch一覧に変換する
bool preparePatches(
    size_t bufferSize,
    const MetadataRegion& metadata,
    const MetadataLayout& layout,
    const std::vector<PatchJob>& jobs,
    std::vector<PreparedPatch>& outPrepared
)
{
    // 出力先を空にしておく
    outPrepared.clear();

    // metadata領域自体がbuffer内にあるか、最初に確認する
    if (!isMetadataInsideBuffer(bufferSize, metadata))
    {
        std::cerr << "metadata region is outside buffer" << std::endl;
        return false;
    }

    for (const PatchJob& job : jobs)
    {
        // 1. fieldNameがlayoutに存在するか確認する
        auto it = layout.find(job.fieldName);

        if (it == layout.end())
        {
            std::cerr << "unknown field: " << job.fieldName << std::endl;
            return false;
        }

        const FieldDescriptor& field = it->second;

        // 2. fieldがmetadata領域内に収まっているか確認する
        if (!isFieldInsideMetadata(metadata, field))
        {
            std::cerr << "field is outside metadata: "
                      << job.fieldName
                      << std::endl;
            return false;
        }

        // 3. 書き換え先がbuffer内に収まっているか確認する
        if (!isPatchDestinationInsideBuffer(bufferSize, metadata, field))
        {
            std::cerr << "patch destination is outside buffer: "
                      << job.fieldName
                      << std::endl;
            return false;
        }

        // 4. .binファイルを読み込む
        std::vector<uint8_t> patchData;

        try
        {
            patchData = readBinaryFile(job.binFilePath);
        }
        catch (const std::exception& e)
        {
            std::cerr << e.what() << std::endl;
            return false;
        }

        // 5. binサイズがfieldサイズと一致するか確認する
        if (patchData.size() != field.size)
        {
            std::cerr
                << "patch size mismatch. field="
                << job.fieldName
                << ", expected="
                << field.size
                << ", actual="
                << patchData.size()
                << std::endl;

            return false;
        }

        // 6. ここまでOKなら、検証済みpatchとして保存する
        PreparedPatch prepared{
            job.fieldName,
            field,
            patchData
        };

        outPrepared.push_back(prepared);
    }

    return true;
}

// 検証済みpatchをbufferへ適用する
bool applyPreparedPatches(
    uint8_t* buffer,
    size_t bufferSize,
    const MetadataRegion& metadata,
    const std::vector<PreparedPatch>& preparedPatches
)
{
    if (buffer == nullptr)
    {
        return false;
    }

    for (const PreparedPatch& patch : preparedPatches)
    {
        // 念のためapply直前にも範囲確認する
        if (!isPatchDestinationInsideBuffer(bufferSize, metadata, patch.field))
        {
            std::cerr << "invalid destination at apply: "
                      << patch.fieldName
                      << std::endl;
            return false;
        }

        const size_t absoluteOffset =
            metadata.baseOffset + patch.field.offset;

        // 対象フィールドだけを上書きする
        std::memcpy(buffer + absoluteOffset,
                    patch.patchData.data(),
                    patch.patchData.size());
    }

    return true;
}

// validate all before apply を実行するメイン関数
bool applyPatchJobsSafely(
    uint8_t* buffer,
    size_t bufferSize,
    const MetadataRegion& metadata,
    const MetadataLayout& layout,
    const std::vector<PatchJob>& jobs
)
{
    std::vector<PreparedPatch> preparedPatches;

    // 1. まず全patchを検証する
    bool valid = preparePatches(
        bufferSize,
        metadata,
        layout,
        jobs,
        preparedPatches
    );

    if (!valid)
    {
        // 検証失敗した場合、bufferは一切変更しない
        return false;
    }

    // 2. 全件OKだった場合のみ、実際にbufferを書き換える
    return applyPreparedPatches(
        buffer,
        bufferSize,
        metadata,
        preparedPatches
    );
}

int main()
{
    try
    {
        // 教材用にpatch用.binを作る
        writeBinaryFile(
            "player_name_patch.bin",
            makeFixedStringPatch("TARO", 16)
        );

        writeBinaryFile(
            "stage_id_patch.bin",
            makeUInt32LittleEndianPatch(42)
        );

        // 実際には巨大ファイルから読み込まれたbufferだと思ってください
        std::vector<uint8_t> buffer(1200, 0x00);

        // metadata領域はbufferの1000バイト目から64バイト分
        MetadataRegion metadata{
            1000,
            64
        };

        // metadataのlayout定義
        MetadataLayout layout{
            {"replay_version", {0, 4}},
            {"player_name",    {4, 16}},
            {"stage_id",       {20, 4}},
            {"recorded_at",    {24, 8}},
            {"reserved",       {32, 32}}
        };

        // 初期状態としてplayer_nameにOLD_NAMEを入れる
        {
            const size_t offset =
                metadata.baseOffset + layout["player_name"].offset;

            const char* oldName = "OLD_NAME";

            std::memcpy(buffer.data() + offset,
                        oldName,
                        std::strlen(oldName));
        }

        // 初期状態としてstage_idに1を入れる
        {
            const size_t offset =
                metadata.baseOffset + layout["stage_id"].offset;

            std::vector<uint8_t> oldStage =
                makeUInt32LittleEndianPatch(1);

            std::memcpy(buffer.data() + offset,
                        oldStage.data(),
                        oldStage.size());
        }

        std::cout << "Before patch - player_name:" << std::endl;
        printBytes(buffer.data(),
                   metadata.baseOffset + layout["player_name"].offset,
                   layout["player_name"].size);

        std::cout << "Before patch - stage_id:" << std::endl;
        printBytes(buffer.data(),
                   metadata.baseOffset + layout["stage_id"].offset,
                   layout["stage_id"].size);

        // 複数のpatch指示
        std::vector<PatchJob> jobs{
            {"player_name", "player_name_patch.bin"},
            {"stage_id",    "stage_id_patch.bin"}
        };

        // 全件検証してから適用する
        bool ok = applyPatchJobsSafely(
            buffer.data(),
            buffer.size(),
            metadata,
            layout,
            jobs
        );

        std::cout << "Patch result: " << ok << std::endl;

        std::cout << "After patch - player_name:" << std::endl;
        printBytes(buffer.data(),
                   metadata.baseOffset + layout["player_name"].offset,
                   layout["player_name"].size);

        std::cout << "After patch - stage_id:" << std::endl;
        printBytes(buffer.data(),
                   metadata.baseOffset + layout["stage_id"].offset,
                   layout["stage_id"].size);
    }
    catch (const std::exception& e)
    {
        std::cerr << "error: " << e.what() << std::endl;
        return 1;
    }

    return 0;
}
```

# ここで何が改善されたか

前回まではこうでした。

```text
1件目を読む
↓
1件目を即適用
↓
2件目を読む
↓
2件目を即適用
```

今回からはこうです。

```text
1件目を読む
2件目を読む
全件サイズチェック
全件範囲チェック
↓
全部OKなら適用
```

つまり、bufferの変更は最後まで遅延されています。

# 失敗例：stage_id_patch.binのサイズが違う場合

例えば `stage_id_patch.bin` が3バイトしかなかったとします。

この場合：

```text
player_name_patch.bin → OK
stage_id_patch.bin    → サイズ不一致
```

前回の設計だと、`player_name` だけ変更済みになってから失敗する可能性がありました。

今回の設計では：

```text
preparePatches() の段階で失敗
↓
applyPreparedPatches() は呼ばれない
↓
bufferは一切変更されない
```

これが大事。

# 今回の思想

今回の考え方は、データ更新処理ではかなり重要です。

```text
検証と適用を分ける
```

英語っぽく言うなら：

```text
Validate first, mutate later.
```

変更処理で一番危険なのは、検証しながら同時に変更することです。

悪い例：

```text
チェックする
↓
書く
↓
チェックする
↓
書く
↓
チェックで失敗
```

良い例：

```text
全部チェックする
↓
全部OK
↓
書く
```

# 今回の授業の結論

複数フィールドをpatchする場合、設計はこうする。

```text
PatchJob
  ユーザー・外部入力に近い差し替え指示

PreparedPatch
  検証済みで、あとはmemcpyするだけの状態

preparePatches()
  全件検証する

applyPreparedPatches()
  全件OKのときだけbufferを書き換える
```

この形にしておくと、事故率がかなり下がります。

次に進むなら、次は **「patch対象が重複している場合」** をやるのが自然です。
例えば、`player_name` と `player_name_short` のように、書き換え範囲が重なっていたらどうするか、です。
