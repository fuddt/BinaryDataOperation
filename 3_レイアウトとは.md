# structとは何か（本質）

まず重要な定義：

> structは「メモリ配置の定義」

例えば：

```cpp
struct A
{
    uint8_t x;
    uint8_t y;
    uint8_t z;
};
```

これはメモリ上で：

```
offset 0 → x
offset 1 → y
offset 2 → z
```

となる

つまり：

```text
struct = layout
```

padding

# paddingとは何か

CPUは：

```
4バイト境界
8バイト境界
```

でアクセスすると速い

だから：

```
uint8_t a;
int b;
```

はこう配置される：

```
offset 0 → a
offset 1
offset 2
offset 3 ← padding
offset 4 → b
```

つまり：

```
合計8バイト
```

---

# offsetofを使う

これはstruct layoutを見る最強ツール

コード：

```cpp
#include <iostream>
#include <cstddef>

struct B
{
    uint8_t a;
    int b;
};

int main()
{
    std::cout << offsetof(B, a) << std::endl;
    std::cout << offsetof(B, b) << std::endl;
}
```

結果：

```
0
4
```

つまり：

```
bはoffset4
```

# 実験：bufferをstructとして読む

コード：

```cpp
#include <iostream>
#include <cstdint>

struct A
{
    uint8_t x;
    uint8_t y;
    uint8_t z;
};

int main()
{
    uint8_t buffer[3] = {10,20,30};

    A* a = reinterpret_cast<A*>(buffer);

    std::cout << (int)a->x << std::endl;
    std::cout << (int)a->y << std::endl;
    std::cout << (int)a->z << std::endl;
}
```

結果：

```
10
20
30
```

つまり：

```
bufferをstructとして読む
```

ことができる
