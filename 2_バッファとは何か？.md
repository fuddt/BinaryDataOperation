# Phase 2: バッファとは何か

バッファとは何か、そしてなぜ `uint8_t*` を使うのかを理解する章。これが Partial Binary Patch の基盤となる。

## Phase 1 の復習

- メモリはバイト列
- 1バイト = 0〜255
- 配列は連続メモリ

本章では、これを「型なし連続メモリとして扱う」ところまで掘り下げる。

---

## Phase2のゴール

この章で理解すべき概念：

① bufferとは何か  
② `uint8_t*` とは何か  
③ `buffer + offset` の意味  
④ なぜ1バイト単位で扱うのか  

これが理解できると `buffer + 12` が完全に読めるようになる。

---

## bufferとは何か

bufferとは「連続したメモリ領域」のこと。

例えば：

```cpp
uint8_t buffer[8] = {
    10,20,30,40,50,60,70,80
};
```

これはメモリ上で：

```
10 20 30 40 50 60 70 80
```

として並ぶ。

---

## bufferの正体

```
uint8_t buffer[8]
```

は内部的に：

```
buffer = &buffer[0]
```

と同じ意味になる。つまり **buffer は先頭アドレス** を指す。

---

## bufferはポインタとして使われる

```cpp
buffer[3]
```

は内部的に：

```cpp
*(buffer + 3)
```

と同じ意味になる。

つまり `buffer + 3` は「3バイト進む」という意味になる。

---

## 実験コード

実行例：

```cpp
#include <iostream>
#include <cstdint>

int main()
{
    uint8_t buffer[8] = {
        10,20,30,40,50,60,70,80
    };

    std::cout << (int)buffer[0] << std::endl;
    std::cout << (int)*(buffer + 0) << std::endl;

    std::cout << (int)buffer[3] << std::endl;
    std::cout << (int)*(buffer + 3) << std::endl;
}
```

出力：

```
10
10
40
40
```

つまり：

```
buffer[index] = *(buffer + index)
```

---

## buffer + offset の正体

```cpp
buffer + 5
```

これは「6番目のバイトのアドレス」を意味する。

---

## アドレスの確認

```cpp
#include <iostream>
#include <cstdint>

int main()
{
    uint8_t buffer[4] = {10,20,30,40};

    std::cout << (void*)&buffer[0] << std::endl;
    std::cout << (void*)&buffer[1] << std::endl;
    std::cout << (void*)&buffer[2] << std::endl;
    std::cout << (void*)&buffer[3] << std::endl;
}
```

出力（例）：

```
0x1000
0x1001
0x1002
0x1003
```

+1 ずつ増えていることが確認できる。

---

## なぜ uint8_t* を使うのか

ポインタの型によって、`p + 1` が進む量が異なる。

```cpp
int* p;
```

の場合、`p + 1` は **4バイト** 進む。

```cpp
uint8_t* p;
```

の場合、`p + 1` は **1バイト** 進む。

`uint8_t*` を使うことで最小単位（1バイト）で移動できる。

### 実験：型の違いによるアドレス差

`int*` の場合：

```cpp
#include <iostream>

int main()
{
    int arr[4];

    std::cout << &arr[0] << std::endl;
    std::cout << &arr[1] << std::endl;
}
```

結果：アドレスが4バイト離れる。

`uint8_t*` の場合：

```cpp
#include <iostream>
#include <cstdint>

int main()
{
    uint8_t arr[4];

    std::cout << (void*)&arr[0] << std::endl;
    std::cout << (void*)&arr[1] << std::endl;
}
```

結果：アドレスが1バイト離れる。

---

## buffer 操作の本質

`uint8_t*` は「バイト単位移動ポインタ」として機能する。

---

## bufferは「意味を持たない」

```cpp
uint8_t buffer[4] = {
    0x64,0x00,0x00,0x00
};
```

このバイト列は `100`（int）かもしれないし、4文字の文字列かもしれないし、浮動小数点の一部かもしれない。

buffer 自体に意味はなく、**意味は後から決まる**。

---

## まとめ

この章で理解すべき概念：

① bufferは連続メモリ  
② bufferは先頭アドレス  
③ `buffer + n` は n バイト進む  
④ `buffer[index]` = `*(buffer + index)`  
⑤ `uint8_t*` は1バイト単位移動
