# bufferとは何か

bufferとは：

```
連続したメモリ領域
```

それ以上でもそれ以下でもない

例えば：

```cpp
uint8_t buffer[8] = {
    10,20,30,40,50,60,70,80
};
```

これはメモリ上で：

```
10 20 30 40 50 60 70 80
```

として並ぶ

---

# bufferの正体

実はこれ：

```
uint8_t buffer[8]
```

は内部的に：

```
buffer = &buffer[0]
```

と同じ意味になる

つまり：

```
bufferは先頭アドレス
```

---

# bufferはポインタとして使われる

例えば：

```cpp
buffer[3]
```

は内部的に：

```cpp
*(buffer + 3)
```

と同じ意味

ここが重要

つまり：

```
buffer + 3
```

は

```
3バイト進む
```

という意味

---

# 実験コードを書く

これを実行する：

```cpp
#include <iostream>
#include <cstdint>

int main()
{
    uint8_t buffer[8] = {
        10,20,30,40,50,60,70,80
    };

    std::cout << (int)buffer[0] << std::endl;
    std::cout << (int)*(buffer + 0) << std::endl;

    std::cout << (int)buffer[3] << std::endl;
    std::cout << (int)*(buffer + 3) << std::endl;
}
```

出力：

```
10
10
40
40
```

つまり：

```
buffer[index]
=
*(buffer + index)
```

---

# buffer + offset の正体

例えば：

```cpp
buffer + 5
```

これは：

```
6番目のバイトのアドレス
```

---

# アドレスを表示して確認する

次のコード：

```cpp
#include <iostream>
#include <cstdint>

int main()
{
    uint8_t buffer[4] = {10,20,30,40};

    std::cout << (void*)&buffer[0] << std::endl;
    std::cout << (void*)&buffer[1] << std::endl;
    std::cout << (void*)&buffer[2] << std::endl;
    std::cout << (void*)&buffer[3] << std::endl;
}
```

出力：

```
0x1000
0x1001
0x1002
0x1003
```

のようになる

つまり：

```
+1ずつ増える
```

---

# なぜ uint8_t* を使うのか

ここが最重要ポイント

例えば：

```cpp
int* p;
```

なら：

```
p + 1
```

は：

```
4バイト進む
```

でも：

```cpp
uint8_t* p;
```

なら：

```
p + 1
```

は：

```
1バイト進む
```

つまり：

```
最小単位で移動できる
```

---

# 実験コード

これを実行する：

```cpp
#include <iostream>

int main()
{
    int arr[4];

    std::cout << &arr[0] << std::endl;
    std::cout << &arr[1] << std::endl;
}
```

結果：

```
4バイト離れる
```

---

次：

```cpp
#include <iostream>
#include <cstdint>

int main()
{
    uint8_t arr[4];

    std::cout << (void*)&arr[0] << std::endl;
    std::cout << (void*)&arr[1] << std::endl;
}
```

結果：

```
1バイト離れる
```

---

# ここがbuffer操作の本質

つまり：

```
uint8_t*
```

は：

```
バイト単位移動ポインタ
```

# bufferは「意味を持たない」

例えば：

```cpp
uint8_t buffer[4] = {
    0x64,0x00,0x00,0x00
};
```

これは

```
100
```

かもしれないし

```
4文字の文字列
```

かもしれないし

```
浮動小数点の一部
```

かもしれない

つまり：

```
意味は後から決まる
```
